# 张雪峰应用：新电脑运营总帮助

## 1. 先理解这套项目是什么
这套项目不是单纯的聊天网页，而是一套“知识库 + 向量检索 + 大模型回答 + 前端咨询界面”的高考志愿咨询系统。

可以把它理解成 5 层：
1. Obsidian/Markdown 内容层：负责存放政策、院校、专业、案例、风格规则。
2. 数据层：07_录取数据 负责历史录取分数、位次、控制线等结构化数据。
3. 向量检索层：Qdrant 负责把知识切块、建立向量索引、按问题检索相关资料。
4. 模型层：
   - embedding 模型负责把问题转成向量
   - chat 大模型负责综合检索结果生成回答
5. 网页层：前端聊天页 + 后台配置页 + Node 后端。

## 2. 在新电脑上，默认只有这份项目文件时，还缺什么
如果对方电脑只有当前项目文件夹，还不能直接完整运行，通常还缺这些：

### 必需安装
1. Node.js 18 或更高版本
用途：运行网页后端、07 抓取脚本、Qdrant 控制台脚本。

2. Qdrant 服务
用途：向量数据库，负责知识检索。
注意：新电脑需要重新安装或重新准备可执行版 Qdrant，不能假设系统里天然存在。

3. LM Studio（如果继续走本地 embedding）
用途：提供本地 embeddings 接口。
如果后面改成云端 embedding，可以不依赖 LM Studio。

### 可能需要重新配置
4. 大模型 API Key
当前项目的导出包里已经去掉密钥。新电脑上需要重新填自己的可用 API Key。

5. embedding 模型
如果继续走本地向量，需要在 LM Studio 里加载可用的 embedding 模型。

## 3. 统一路径写法
以后不要再写死 `D:\张雪峰应用` 这种本机路径，统一按“项目根目录”理解。

本文里所有路径都按下面这个规则看：
- `【项目根目录】` = 对方电脑解压后最外层项目目录

例如：
- `【项目根目录】\高考仓库\13_网页前端`
- `【项目根目录】\高考仓库\14_qdrant`
- `【项目根目录】\高考仓库\07_录取数据`

## 4. 目录分工

### 00_项目总控
全局说明、总览、地图、图谱入口。

### 01_政策规则
各省政策、赋分、新高考模式、提前批、专项计划、特殊类型招生等。

### 02_省份数据
各省录取数据总览页、规则入口、历史数据提醒、官方查询入口。

### 03_院校库
院校页、分省总表、院校定位、报考风险、适合什么学生。

### 04_专业库
专业定义、优缺点、未来前景、家庭约束、发展路线、适合的学校层级。

### 05_张雪峰风格库
表达风格、家庭约束标签、情绪沟通模板、FAQ、判断框架。

### 06_案例库
典型咨询场景、捡漏案例、家庭约束案例、冲稳保分析案例。

### 07_录取数据
结构化历史数据，包括：
- 各省控制线
- 各院校在各省录取分数线
- 各专业在各省录取分数线
- 位次、批次、科类等信息

### 13_网页前端
首页、问答页、后台配置页、Node 后端入口。

### 14_qdrant
Qdrant 相关配置、collection 模板、路由脚本、控制台前端。

## 5. 整个系统运行逻辑
用户提问时，系统不是直接把问题丢给大模型，而是按下面流程走：

1. 用户在网页输入问题。
2. Node 后端先判断问题类型。
3. 后端按问题类型选择应该查哪个知识分区。
4. embedding 模型把问题转成向量。
5. Qdrant 根据向量检索最相关的资料片段。
6. 后端把检索结果和系统提示词一起交给大语言模型。
7. 大模型生成最终回答。
8. 前端显示回答、来源、进度状态。

## 6. 最重要的路由原理
这套系统不是“全库乱搜”，而是先做问题分类，再决定查哪些集合。

### 主要路由
1. `school_consult`
适用于：
- 介绍某学校
- 对比两所学校
- 问某学校值不值得报

优先查：
- 院校库
- 少量相关案例

2. `major_consult`
适用于：
- 某专业前景如何
- 某专业适不适合自己
- 哪个专业更值

优先查：
- 专业库
- 风格库
- 案例库

3. `policy_consult`
适用于：
- 某省提前批怎么报
- 赋分规则是什么
- 专项计划、公费师范、军警等政策问题

优先查：
- 政策规则
- 省份数据总览

4. `score_tendency`
适用于：
- 某分数/位次能不能上某学校
- 某专业大概是冲还是稳

优先查：
- 07_录取数据
- 省份数据总览
- 必要时补学校页和专业页

5. `family_emotion`
适用于：
- 家里条件一般怎么选
- 想尽早赚钱怎么办
- 分数不理想如何安抚并给方案

优先查：
- 风格库
- 案例库
- 专业库

### 路由原则
- 明确学校名：优先走学校路由
- 明确专业名：优先走专业路由
- 明确省份 + 政策关键词：优先走政策路由
- 明确分数/位次/录取：优先走录取倾向路由
- 明确家庭压力/情绪问题：优先走家庭与情绪路由

## 7. 07_录取数据 现在能做什么
07 目前已经可以支撑“简单历史倾向判断”，但不要把它当成当年绝对结论。

### 07 适合做的事情
- 判断某学校/专业历史门槛高不高
- 根据历史最低分、最低位次做“冲 / 可搏 / 偏稳 / 高风险”判断
- 看各省控制线、批次线

### 07 不适合直接说死的话
- “你今年一定能上”
- “这个学校稳了”
- “今年肯定录你”

### 最稳的输入方式
用户最好提供：
- 省份
- 分数或位次
- 科类/选科
- 学校或专业目标

其中：
- **位次优先于分数**
- 分数只能做粗判断，位次更稳

## 8. 新电脑运行前要先准备什么

### A. 安装 Node.js
验证命令：
`node -v`

### B. 安装并启动 Qdrant
如果项目包里没有现成可执行版，需在新电脑安装 Qdrant。
启动后应能访问：
- `http://127.0.0.1:6333/`

### C. 如果继续走本地向量
安装并启动 LM Studio，加载 embedding 模型。
启动后应能访问：
- `http://127.0.0.1:1234/v1/embeddings`

### D. 配置大模型 API
在后台页面填入自己的：
- provider
- base URL
- model
- API key

## 9. 新电脑最稳的启动顺序
1. 启动 LM Studio（如果要本地 embedding）
2. 启动 Qdrant
3. 打开项目根目录下的一键启动脚本
4. 打开后台页检查配置
5. 再打开前台问答页测试

## 10. 对方最常用的脚本
假设对方拿到的是整理后的轻量包，最常用的是：
- `【项目根目录】\一键启动_局域网前后端.bat`
- `【项目根目录】\一键停止_局域网前后端.bat`
- `【项目根目录】\开始抓分数.bat`

### 说明
- 启动/停止脚本已经改成相对路径，只要保留完整目录结构，解压到任意盘符都能运行。
- `开始抓分数.bat` 也已经改成相对路径，会在缺少 07 依赖时自动执行 `npm install`。

## 11. 最容易坏的地方
1. 没有安装 Node.js
2. 没有启动 Qdrant
3. 没有启动 LM Studio embeddings
4. 后台 API Key 没填
5. Qdrant collection 没有准备好
6. 把目录拆散，导致脚本相对路径失效

## 12. 交接时最重要的一句话
这套系统不是“网页直接问模型”，而是：
**先路由，再向量检索，再让大模型结合知识库回答。**

如果检索层坏了，回答质量会明显下降；如果模型层坏了，网页会直接无法返回答案；如果数据层不完整，系统只能给出历史倾向判断，不能给出当年绝对结论。
